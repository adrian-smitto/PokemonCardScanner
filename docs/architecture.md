## Hash Database (`db/cards.db`)

Built once by `build_db.py`. Never modified at scan time.

```sql
CREATE TABLE cards (
    id        TEXT PRIMARY KEY,  -- e.g. "sv6-123"
    name      TEXT NOT NULL,
    set_name  TEXT NOT NULL,
    set_id    TEXT NOT NULL,
    number    TEXT NOT NULL,
    image_url TEXT NOT NULL,
    phash     TEXT NOT NULL,     -- 64-char hex (256-bit phash, hash_size=16)
    rarity    TEXT
);
CREATE INDEX idx_phash ON cards(phash);
```

**Build process**: Paginate all cards from `GET /v2/cards` (250/page, ~15‚Äì18k total) ‚Üí download `smallImage` URLs via `ThreadPoolExecutor(8)` ‚Üí compute `imagehash.phash(img, hash_size=16)` ‚Üí batch-insert. Resumable (skips existing IDs). Expect 20‚Äì40 min on first run.

## Scanning State Machine

```
IDLE ‚Üí CARD_DETECTED ‚Üí STABILIZING ‚Üí HASHING ‚Üí MATCHING ‚Üí FETCHING_PRICE ‚Üí DONE ‚Üí COOLDOWN ‚Üí IDLE
```

| State | Trigger to advance | Notes |
|---|---|---|
| IDLE | Card contour detected | Waiting for card |
| CARD_DETECTED | N stable frames | Contour found but not yet stable |
| STABILIZING | Consecutive frame hashes agree (hamming < threshold) | Prevents triggering mid-placement |
| HASHING | Hash computed | Canonical hash = median of stable frames |
| MATCHING | DB lookup complete | Full-table Python hamming scan (~50‚Äì150ms) |
| FETCHING_PRICE | API returns (async, ThreadPoolExecutor) | Scanning can continue; result via queue |
| DONE | Result emitted to queue | Beep plays |
| COOLDOWN | Timer expires OR card removed | Suppresses duplicate scans (default 5s) |

## Thread Model

```
Main Thread (Tkinter)
  ‚îî‚îÄ‚îÄ _tick() every 33ms
        ‚îú‚îÄ‚îÄ reads CameraCapture.latest_frame
        ‚îú‚îÄ‚îÄ drives StateMachine.process()
        ‚îî‚îÄ‚îÄ drains result_queue ‚Üí updates UI

Camera Thread (daemon)
  ‚îî‚îÄ‚îÄ CameraCapture._capture_loop()
        ‚îî‚îÄ‚îÄ writes latest_frame under threading.Lock

Worker ThreadPoolExecutor (max 2 workers)
  ‚îú‚îÄ‚îÄ hash matching (CPU-bound, ~50‚Äì150ms)
  ‚îî‚îÄ‚îÄ price API fetch (IO-bound, ~200‚Äì800ms)

Audio Thread (fire-and-forget daemon)
  ‚îî‚îÄ‚îÄ winsound.Beep()
```

**Rule**: All Tkinter widget updates happen exclusively in `_tick()` on the main thread. Worker threads communicate only via `queue.Queue`.

## Duplicate Scan Prevention

1. **COOLDOWN state** ‚Äî after a successful scan the machine ignores new detections for `DUPLICATE_COOLDOWN_SECONDS` (default 5s)
2. **Hash drift guard** ‚Äî if the frame hash drifts significantly during COOLDOWN (card removed), transition to IDLE immediately ‚Äî enables fast scanning without waiting for the timer
3. **Session ID-based guard** ‚Äî secondary safety net in `AppWindow`

## Implementation Order

1. **`build_db.py`** ‚Äî build and validate the hash DB first; manually photograph ~10 cards to verify match accuracy
2. **Core pipeline** ‚Äî `camera.py` ‚Üí `detector.py` ‚Üí `cropper.py` ‚Üí `hasher.py` ‚Üí `matcher.py` as a standalone terminal script; tune detection thresholds
3. **`state_machine.py` + `price_client.py` + `audio.py`** ‚Äî full scanning pipeline without UI
4. **UI** ‚Äî `feed_panel.py` ‚Üí `result_panel.py` ‚Üí `log_panel.py` ‚Üí `app_window.py`
5. **`scan_log.py`** ‚Äî persistence, CSV export, session stats
